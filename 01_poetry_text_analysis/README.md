# Сбор и очистка датасета русской поэзии

> Проект для портфолио продуктового аналитика.  
> Демонстрирует навыки сбора, очистки и анализа данных.

---

## О проекте

**Цель:** Собрать и подготовить корпус русскоязычных стихотворений для последующего анализа и машинного обучения.

**Что сделано:**
- Разработан парсер для сбора стихов с сайта rustih.ru
- Проведена многоступенчатая предобработка данных
- Выполнен квантильный анализ для фильтрации по длине
- Проведён визуальный анализ частоты авторов и распределения длины текстов
- Сформирован структурированный датасет из 46 467 стихотворений

---

## Источник данных

**Сайт:** [rustih.ru](https://rustih.ru)

**Охват:** 46 467 стихотворений, более 140 авторов, классические и современные жанры и эпохи

---

## Стек технологий

| Инструмент | Назначение |
|------------|------------|
| **Scrapy** | Асинхронный парсинг |
| **BeautifulSoup** | Удаление HTML-тегов |
| **Pandas** | Обработка и фильтрация данных |
| **Matplotlib / Seaborn** | Визуализация распределений |
| **Re (regex)** | Удаление управляющих символов |

---

## Парсинг данных

**Процесс:**
1. Двухуровневый парсинг: сначала список поэтов, затем стихотворения каждого автора
2. Очистка от рекламных баннеров и прозаических комментариев
3. Сохранение в CSV с полями: `Text`, `Author`, `Title`, `URL`

---

## Предобработка данных

### Многоступенчатая очистка

**Шаг 1. Очистка текста (функция `clean_text`)**
- Удаление HTML-тегов (BeautifulSoup)
- Удаление управляющих символов
- Удаление избыточных пробелов

**Шаг 2. Предобработка (функция `preprocess_data`)**
- Удаление пустых строк
- Удаление пропусков значений
- Удаление дублирующихся стихотворений

**Шаг 3. Квантильный анализ для фильтрации по длине**
- Вычислены нижний (0.05) и верхний (0.95) квантили
- Удалено 5% стихотворений с наибольшим количеством символов
- Удалено 5% стихотворений с наименьшим количеством символов
- Сохранено 90% наиболее репрезентативных текстов
